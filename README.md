# Silent Signal — A Privacy-Preserving Dead Man's Switch on Midnight

Silent Signal is a zero-knowledge smart contract built with Compact on the [Midnight Network](https://midnight.network). It lets someone lock a secret (represented as a cryptographic hash) on-chain, and prove ownership of it privately through periodic "check-ins" — without ever revealing their identity or the secret itself on-chain.

If the owner stops checking in for a set number of rounds (simulating a real-world time-based grace period), **anyone** can trigger a public release of the secret. This models real-world use cases like whistleblower protection, investigative journalism "insurance files," or digital inheritance — where information should only become public if something has gone wrong.

## Why This Idea

Most beginner Web3 demos are counters or voting apps. Silent Signal instead demonstrates a genuinely useful pattern unique to privacy-preserving chains like Midnight: **prove something without revealing it, until the right moment.**

### Known limitations & how they're addressed

| Challenge | Solution used in this contract |
|---|---|
| Blockchains have no native concept of "time" | Uses on-chain `round` counters (via the ledger's `Counter` type) instead of wall-clock time |
| "Automatic" release needs a trigger | `checkAndRelease` is a **permissionless** circuit — anyone can call it, but it only succeeds once the grace period has genuinely elapsed |
| Large secrets can't live on-chain | Only a **hash** of the secret is stored on-chain; the real content would live off-chain (e.g. IPFS), decrypted only once the hash-gated release happens |
| Accidental false triggers | A grace period (`3` rounds) must fully elapse with no check-in before release is possible |
| Legal/ethical concerns of "auto-leaking" data | This is a technical proof-of-concept only. A production version would need legal review, content policies, and user consent flows |

## Public State vs. Private Witness

This is the core privacy concept the contract demonstrates.

- **Public ledger state** (visible to everyone on-chain): `authority`, `secretHash`, `state`, `round`, `lastCheckInRound`. Anyone can see that a secret is locked, and its current status — but not who owns it or what the secret is.
- **Private witness** (`secretKey`): the owner's raw secret key, which **never leaves their local machine**. It's only used locally to compute a zero-knowledge proof.
- **`disclose()`**: Compact requires this explicit keyword any time a value derived from private data is written to public state. This makes accidental privacy leaks a compile-time error, not a runtime bug.

In this contract, the owner's secret key is never revealed — only a one-way hash "fingerprint" (`publicKey`) of it is disclosed, so the contract can verify future check-ins came from the same owner, without ever knowing who that owner is.

## Setup Instructions (Run Locally)

```bash
# 1. Clone this repository
git clone <this-repo-url>
cd <repo-folder>

# 2. Install dependencies
yarn install

# 3. Compile the contract
yarn compile

# 4. Start the local devnet (Docker)
yarn env:up

# 5. Run the test suite against the local devnet
yarn test:local

# 6. Stop the local devnet when done
yarn env:down
```

## Deploying to Preview Testnet

```bash
# 1. Generate a Midnight wallet (e.g. via the Lace browser extension)
# 2. Fund it with tNIGHT from the Preview faucet, then generate tDUST in Lace
# 3. Create .env.preview with your wallet mnemonic:
#    MIDNIGHT_PREVIEW_MNEMONIC="your 24 word mnemonic here"
# 4. Start the proof server
yarn proof:up

# 5. Load the env file and deploy
set -a && source .env.preview && set +a
yarn test:preview
```

## How It Works

1. **`lockSecret(hash)`** — the owner locks a secret's hash, and their private key's fingerprint becomes the on-chain `authority`
2. **`checkIn()`** — only the real owner (proven via their private witness) can update `lastCheckInRound`, resetting the grace-period clock
3. **`checkAndRelease()`** — callable by **anyone**, but only succeeds once `round - lastCheckInRound >= 3` (the grace period)
4. **`getSecretHash()`** — once released, anyone can read the secret's hash publicly
5. **`tick()`** — a test-only helper to simulate round progression (in production this would be driven by real block progression)

## Test Results

All 5 tests pass, covering the full lifecycle: deploy → lock → check-in → premature-release rejection → successful release after the grace period.
